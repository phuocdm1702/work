Git Flow Best Practices

Luồng làm việc chuẩn (Team Workflow)

Issue/Task
feature/abc
Draft PR + Label
Code & Push
Ready for Review
Merge

Nguyên tắc: Tạo PR sớm + Gắn nhãn để team biết bạn đang làm task nào

Nhận Task → Tạo nhánh → Tạo Draft PR gắn nhãn "In Progress" → Code → Ready → Review → Merge

1

Nhận Task & Tạo Nhánh

Khi được giao task/issue, tạo nhánh ngay và push lên để team theo dõi:

# 1. Checkout develop và pull code mới nhất
git checkout develop git pull origin develop
# 2. Tạo feature branch từ develop (đặt tên rõ ràng)
# Format: feature/{issue-id}-{mo-ta-ngan} git checkout -b feature/123-login-page git checkout -b feature/456-fix-api-timeout
# 3. Push nhánh lên GitHub ngay (để team thấy)
git push -u origin feature/123-login-page
2

Tạo Pull Request Sớm + Gắn Nhãn (Quan trọng)

Tạo PR ngay sau khi tạo nhánh, gắn nhãn In Progress để team biết bạn đang làm task này:

Các bước thực hiện:

  1. Truy cập GitHub → Click "Compare & pull request"
  2. Chọn base: develop
  3. Tiêu đề: [WIP] Feature: Login page implementation
  4. Mô tả ngắn: Link issue, mô tả những gì sẽ làm
  5. Gắn nhãn: In Progress hoặc Draft
  6. Gán assignee: Bản thân bạn
  7. Click "Create pull request"

Nhãn thường dùng:

In Progress - Đang code
Ready for Review - Chờ review
Approved - Đã approve
Needs Changes - Cần sửa

Lợi ích:

  • • Team biết ai đang làm task nào
  • • Tránh 2 người làm cùng task
  • • Có thể xem progress qua commits
  • • Dễ dàng review sớm nếu cần
3

Commit thay đổi

Lưu các thay đổi với message chuẩn Conventional Commits:

# Thêm file vào staging
git add filename.js hoặc git add . (thêm tất cả)
# Commit với message chuẩn
git commit -m "feat(auth): add JWT authentication" git commit -m "fix(api): resolve null pointer exception"
# Xem lịch sử commit
git log --oneline --graph --decorate

Các loại commit:

feat: Tính năng mới
fix: Sửa lỗi
docs: Tài liệu
test: Test case
refactor: Cải tiến
chore: Bảo trì
4

Push & Pull Request

Đưa code lên GitHub và tạo Pull Request để review:

# Push lần đầu (thiết lập upstream)
git push -u origin feature/login-page
# Push các commit tiếp theo
git push
# Cập nhật branch với develop (rebase)
git fetch origin git rebase origin/develop

Sau khi push:

  1. Truy cập repository trên GitHub
  2. Click "Compare & pull request"
  3. Chọn base branch: develop
  4. Viết mô tả rõ ràng về thay đổi
  5. Gán reviewer và tạo PR
  6. Sau khi approve: merge vào develop
5

Merge & Dọn dẹp

Sau khi PR được merge, dọn dẹp branch local:

# Chuyển về develop
git checkout develop
# Pull code mới sau merge
git pull origin develop
# Xóa branch local đã merge
git branch -d feature/login-page
# Xóa branch remote đã merge
git push origin --delete feature/login-page

Lệnh tham khảo nhanh

Branch

git branch Liệt kê branches
git branch -a Tất cả branches
git checkout -b name Tạo & chuyển branch

Undo

git restore file Bỏ thay đổi chưa stage
git reset --soft HEAD~1 Bỏ commit giữ changes
git revert commit-id Revert commit

Quy ước đặt tên

feature/login-page
fix/api-timeout
hotfix/critical-bug
release/v1.2.0

Tình huống thường gặp

Conflict khi merge

git status # Sửa file conflict git add . git commit -m "fix: resolve merge conflict"

Quên commit message

git commit --amend

Stash tạm thởi

git stash # Làm việc khác... git stash pop

Best Practices

  • Commit thường xuyên, mỗi commit 1 logic
  • Viết message rõ ràng, có prefix
  • Pull trước khi push
  • Xóa branch sau khi merge
  • Không commit trực tiếp lên main
