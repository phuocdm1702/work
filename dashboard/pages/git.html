Checklist thao tác
  1. 1
    Đảm bảo main sạch và mới nhất
    git checkout main
    git pull origin main
  2. 2
    Tạo nhánh mới cho buổi/tính năng
    git checkout -b feat/lesson-03
  3. 3
    Code → add → commit (nhỏ, rõ ràng)
    git status
    git add .
    git commit -m "feat: add lesson 03 exercises"
  4. 4
    Push nhánh lên GitHub
    git push -u origin feat/lesson-03
  5. 5
    Tạo Draft PR + gắn label in progress
    Trên GitHub (Web):
    1) Mở repo → tab Pull requests → New pull request
    2) Chọn base: main và compare: feat/lesson-03
    3) Click Create pull request → tick Create as draft
    4) Sau khi PR tạo xong: bên phải chọn Labels → gắn in progress
  6. 6
    Cập nhật PR
    Mỗi lần commit + push thêm là PR tự cập nhật.
  7. 7
    Merge → dọn nhánh
    git checkout main
    git pull origin main
    git branch -d feat/lesson-03
Ngoại lệ thường gặp (và cách xử lý)
Đang làm dở mà cần chuyển nhánh gấp
git stash

Pull bị conflict
Mở file có marker <<<<<<< / ======= / >>>>>>> → chọn đúng code → add → commit.
git status
git add <file>
git commit
Lỡ commit sai message / muốn sửa commit gần nhất
git commit --amend
Lỡ commit lên main (chưa push)
git reset --soft HEAD~1
git checkout -b feat/fix-mistake
git commit -m "fix: move changes off main"
Tạo PR bị báo lỗi: PR đã tồn tại
Nguyên nhân: đã có PR mở từ nhánh đó vào main.
Cách 1 (khuyên dùng): mở PR cũ → tiếp tục commit + push lên cùng nhánh, PR sẽ tự cập nhật.
Cách 2: nếu PR cũ sai mục tiêu → Close PR cũ → tạo PR mới từ nhánh đúng.
Không cần tạo PR mới mỗi lần push. 1 PR = 1 nhánh (thường là vậy).
Nhắc nhanh
Trước khi tạo nhánh mới
Luôn `pull main` để tránh conflict về sau.
Trước khi merge
Đảm bảo PR xanh (không lỗi) và mô tả rõ thay đổi.

Giải mã các Flags (Options) phổ biến

Lệnh checkout / switch

  • -b Branch: Tạo nhánh mới và chuyển sang đó ngay lập tức.
  • -c Create: (Trong lệnh `git switch`) Tương đương `-b` của checkout.

Lệnh commit

  • -m Message: Viết nội dung commit trực tiếp trên terminal.
  • -am Add & Message: Vừa `add` (cho file đã track) vừa `commit` một lúc.

Lệnh push / remote

  • -u Upstream: Thiết lập mối quan hệ theo dõi giữa nhánh local và remote.
  • -f Force: Ép buộc đẩy code lên (CỰC KỲ NGUY HIỂM).

Lệnh branch

  • -d Delete: Xóa một nhánh (chỉ xóa khi đã merge).
  • -D Force Delete: Xóa nhánh bất kể đã merge hay chưa.
